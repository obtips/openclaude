# Prompt 工程技巧

编写优秀的 Prompt 是与 AI 有效沟通的关键。本教程将分享实用的 Prompt 工程技巧。

## 核心原则

### 1. 清晰具体

**差的 Prompt：**
```
帮我写个函数
```

**好的 Prompt：**
```
请用 Python 写一个函数，接收一个字符串列表，返回按字母顺序排序后的列表。
如果列表为空，返回空列表。包含类型注解和简单的 docstring。
```

### 2. 提供上下文

```
我正在开发一个 Django 电商网站，需要实现购物车功能。
请帮我设计购物车的数据模型，包括以下功能：
- 添加商品
- 移除商品
- 更新数量
- 计算总价

使用 Django 的 ORM，并考虑库存管理。
```

### 3. 明确输出格式

```
请分析以下代码的性能问题：

[代码...]

请按以下格式输出：
1. 问题识别：列出发现的问题
2. 影响：说明每个问题的影响
3. 建议：提供具体的优化方案
4. 示例：给出优化后的代码
```

## 高级技巧

### 角色扮演

```
你是一位有 10 年经验的高级 Python 工程师，
精通 Django、FastAPI 和异步编程。
请审查以下代码，并提出改进建议：
```

### 思维链 (Chain of Thought)

```
让我们一步步解决这个问题：

第一步：分析问题的核心需求...
第二步：列出可能的解决方案...
第三步：评估每种方案的优劣...
第四步：选择最佳方案并实现...

现在，请按照这个思路来解决以下问题...
```

### 少样本学习 (Few-Shot Learning)

```
示例 1：
输入：今天天气真好
输出：正面情感

示例 2：
这个产品质量太差了
输出：负面情感

示例 3：
还可以吧
输出：中性情感

现在请判断：
这家餐厅的菜品味道不错，但价格有点贵
输出：
```

## 常见场景

### 代码审查

```
请从以下角度审查这段代码：
1. 代码风格和可读性
2. 性能问题
3. 潜在的 bug
4. 安全性考虑
5. 测试建议

代码：
[你的代码]
```

### 技术写作

```
请为以下 API 端点编写文档：
- 功能描述
- 请求参数（类型、是否必需、说明）
- 响应格式
- 使用示例
- 错误处理

API 端点：GET /api/users/{id}
```

### 调试辅助

```
我遇到了以下错误：

[错误信息]

相关代码：
[代码片段]

请帮我：
1. 分析错误原因
2. 提供修复方案
3. 解释如何避免类似问题
```

## 避免的陷阱

### 1. 过于宽泛

❌ `告诉我关于 JavaScript 的一切`

✅ `解释 JavaScript 中的闭包概念，并给出一个实际应用场景`

### 2. 缺少上下文

❌ `这个函数有什么问题？`
（没有提供代码）

✅ `请审查以下函数...`（附上代码和使用场景）

### 3. 一次性要求太多

❌ `帮我构建一个完整的电商网站，包括前后端、数据库、支付、物流...`

✅ `我们先从用户认证模块开始，使用 JWT 实现登录功能`

## 实用模板

### 学习新概念

```
请用以下方式解释 [主题]：
1. 简单定义（一句话）
2. 核心概念（3-5 点）
3. 实际应用示例
4. 与 [相关概念] 的区别
5. 推荐的学习资源

面向有 [背景] 的初学者。
```

### 代码重构

```
请重构以下代码，目标是：
1. 提高可读性
2. 降低复杂度
3. 遵循 [语言/框架] 最佳实践

原代码：
[代码]

重构后请：
- 保持相同的功能
- 添加注释说明改动原因
- 指出改进的具体点
```

### 问题诊断

```
问题描述：
[详细描述问题]

环境信息：
- 语言/框架版本
- 运行环境
- 相关配置

已尝试的解决方案：
1. [方案1] - 结果
2. [方案2] - 结果

请帮我：
1. 分析可能的原因
2. 提供诊断步骤
3. 给出解决方案
```

## 进阶：迭代优化

优秀的 Prompt 往往需要多次迭代：

```
# 第一版
帮我写个爬虫

# 第二版
帮我用 Python 写一个网页爬虫，使用 BeautifulSoup
# 输出：基础的爬虫代码

# 第三版
在上面的基础上，添加：
- 请求头伪装
- 随机延迟
- 错误重试机制
- 日志记录
# 输出：更健壮的爬虫

# 第四版
请进一步优化，使其可以：
- 支持多线程
- 限速控制
- 数据持久化到数据库
# 输出：生产级的爬虫
```

## 总结

好的 Prompt 应该：

- ✅ **具体明确** - 说清楚你想要什么
- ✅ **提供上下文** - 给出足够的背景信息
- ✅ **指定格式** - 明确期望的输出格式
- ✅ **循序渐进** - 复杂任务分解成步骤
- ✅ **迭代优化** - 根据结果不断改进

记住：Prompt 工程是一门实践的艺术，多尝试、多调整！
