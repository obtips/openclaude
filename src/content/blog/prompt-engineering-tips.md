## 为什么 Prompt 质量很重要

与 Claude 交流时，提示词的质量直接影响输出结果。一个精心设计的 prompt 可以让 Claude 更准确地理解你的需求。

## 技巧 1：明确具体的指令

不要说："帮我写一篇文章"
要说："帮我写一篇关于人工智能的 1000 字技术文章，面向普通读者"

**原因**: 明确的指令可以减少歧义，提高输出质量。

## 技巧 2：提供上下文信息

```
我正在开发一个 Python Web 应用，使用 Flask 框架。
请帮我设计一个用户认证系统的架构。
```

**原因**: 上下文信息帮助 Claude 更好地理解场景和约束条件。

## 技巧 3：指定输出格式

```
请以表格形式展示以下数据对比，包含以下列：
- 特性名称
- 方案 A
- 方案 B
- 优劣分析
```

**原因**: 指定格式可以减少后续调整的工作量。

## 技巧 4：使用示例引导

```
输入: "今天天气真好"
输出: "今天天气真好 ☀️"

请按照上述示例为以下句子添加合适的 emoji：
输入: "我很开心"
输出: ?
```

**原因**: 示例是展示期望效果的最直接方式。

## 技巧 5：分步骤处理复杂任务

对于复杂任务，将其拆分为多个步骤：

```
第一步：分析需求，列出关键要点
第二步：根据要点生成大纲
第三步：填充内容
```

**原因**: 分步骤可以降低任务复杂度，提高准确性。

## 总结

好的 prompt 应该：
- ✅ 明确具体
- ✅ 包含上下文
- ✅ 指定格式
- ✅ 提供示例
- ✅ 分步骤处理

持续练习和迭代是提升 prompt 编写能力的关键！
